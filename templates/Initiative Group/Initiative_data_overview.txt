1. MAINTAINABILITY
Initiative: Maintainability Enhancements
Key Epics/Features: Tech Debt & UI improvements across teams (ATH, EOS, SPA, HEL, DAID), Technology Upgrades, Config Items Management.
Relevant IRMs: IRM-40177, IRM-38242, IRM-37610, IRM-33555, IRM-33531, IRM-30464, IRM-29268.
Why it's important: Focuses heavily on reducing technical debt, restructuring the revenue-management-be-custom to hexagonal architecture, and crucial technology upgrades (Java 25, Spring Boot 4.x.x, Oracle 26ai). It also introduces a highly requested Config Items Management tool to safely package and migrate configurations across environments.
2. ACCESSIBILITY
Initiative: Accessibility Enhancements
Key Epics/Features: Accessibility Improvements across all teams.
Relevant IRMs: IRM-40171, IRM-40170, IRM-40169, IRM-40167, IRM-39187, IRM-30421.
Why it's important: Ensures the Solon Tax application is strictly compliant with WCAG 2.1 AA and EN 301 549 standards. This includes keyboard navigation, screen reader support, and color contrast fixes.
3. USABILITY
Initiative: Usability Enhancements
Key Epics/Features: UI/UX Redesigns, AI Integration, Search Experience Improvements.
Relevant IRMs: IRM-40181, IRM-39984, IRM-39979, IRM-38784, IRM-30357, IRM-30358, IRM-30419.
Why it's important: Major focus on user experience, including a new "Solon Look and Feel" (Figma-based redesigns), better global search panels, and the introduction of AI in both RM and Self-Service (e.g., chatbots, natural language navigation, and anomaly detection).
4. PERFORMANCE EFFICIENCY
Initiative: Performance Efficiency Enhancements
Key Epics/Features: Green Coding, DB Partitioning, Performance Suite Enrichment.
Relevant IRMs: IRM-39738, IRM-39200, IRM-38238, IRM-34521, IRM-34169, IRM-31562.
Why it's important: Targets high-volume operations. Includes partitioning the database for volume management, optimizing CI/CD pipelines for "Green Coding" (reducing resource footprint), and enhancing caching for the Reference Data Microservice.
5. TAXPAYER ACCOUNTING
Initiative: Taxpayer Accounting Enhancements
Key Epics/Features: Financial events logging, GL Decisions Table enhancements.
Relevant IRMs: IRM-40159, IRM-39736, IRM-37844, IRM-36179, IRM-30362.
Why it's important: Expands the GL decision table to support multiline rules and additional parameters (Valid From/To, Taxpayer attributes), making accounting configurations highly scalable and reducing the need for excessive Financial Transaction types.
6. USER & 3RD PARTY ACCESS MANAGEMENT
Initiative: User & 3rd Party Access Management Enhancements
Key Epics/Features: Representations (Delegations, 2-way/multilevel authorizations), 3rd Party System Support.
Relevant IRMs: IRM-39135, IRM-38190, IRM-36666, IRM-36665, IRM-32538.
Why it's important: Radically improves how taxpayers delegate rights (Representations). It supports multilevel delegations, captures metadata like "Source System," and extends the data model to integrate securely with external/3rd-party systems.
7. SECURITY
Initiative: Security Enhancements
Key Epics/Features: GDPR Data Security, Row-Level Security, Keycloak Active Cluster.
Relevant IRMs: IRM-39131, IRM-39113, IRM-38847, IRM-36809, IRM-31569, IRM-20329.
Why it's important: Implements strict GDPR compliance by allowing admins to mask, encrypt, or restrict visibility of sensitive personal data based on classification levels. Also ensures high availability by moving Keycloak to an active cluster mode.
8. TAX ACCOUNTS
Initiative: Tax Accounts Enhancements
Key Epics/Features: Excel Exporting, Threshold Amounts.
Relevant IRMs: IRM-39021, IRM-38899, IRM-38308.
Why it's important: Adds quality-of-life features like exporting fully filtered GL Accounts to Excel (bypassing UI pagination limits) and introducing threshold amounts in Revenue Type Registrations.
9. VAT
Initiative: Reference Implementation - VAT
Key Epics/Features: VAT Tax Account Reference Implementation.
Relevant IRMs: IRM-38759.
Why it's important: Provides a reference implementation to demo adding debts and payments specifically to a VAT tax account.
10. UI CONFIGURATION
Initiative: UI Configuration Enhancements
Key Epics/Features: Configurable Home Page Widgets, Visual Cues.
Relevant IRMs: IRM-38680, IRM-32420, IRM-30418.
Why it's important: Empowers configurators to design customizable RM home pages with widgets (e.g., pending tasks, team calendars) and adds UI schema support to highlight required vs. optional fields visually.
11. FORM PROCESSING
Initiative: Form Processing Enhancements
Key Epics/Features: Drools Orchestration in SS, Prefill vs. Validation separation.
Relevant IRMs: IRM-38379, IRM-37656, IRM-34563, IRM-30402.
Why it's important: Enables full Drools process orchestration directly in Self Service (SS) for validations and calculations. It also decouples form prefill logic from validation logic for better API flexibility.
12. TRACEABILITY
Initiative: Traceability Enhancements
Key Epics/Features: Audit Logs for Codelists/Registration/Authorizations, Context Records.
Relevant IRMs: IRM-38336, IRM-38335, IRM-32746, IRM-30369, IRM-17620.
Why it's important: Vastly improves the Audit Trail by grouping logs logically (e.g., tracking all changes related to a specific Declaration ID/Form) and tracking modifications to critical dates like the Statute of Limitation.
13. REFERENCE DATA
Initiative: Reference Data Enhancements
Key Epics/Features: Import/Export Code Lists, Translatable Metadata, Validity Periods.
Relevant IRMs: IRM-38290, IRM-36980, IRM-33364, IRM-29344, IRM-18730, IRM-18729.
Why it's important: Makes managing reference data easier by allowing XML import/export of code lists. Also adds robust localization support so codelist attributes and metadata can be translated into multiple languages.
14. REGISTRATION
Initiative: Registration Enhancements
Key Epics/Features: Organizational/Taxpayer Hierarchy Trees, Backdated Changes, Expanded Name Elements.
Relevant IRMs: IRM-37845, IRM-37212, IRM-36002, IRM-31650, IRM-30371.
Why it's important: Introduces graphical tree views for complex organizational and taxpayer hierarchies. It also allows officers to view a party's "Details in Time" (historical backdated changes) and expands name elements to support legacy data migrations.
15. BILLING, ASSESSMENTS AND INSTALLMENTS
Initiative: Billing & Assessments Enhancements
Key Epics/Features: Ad-hoc Obligations & Installments.
Relevant IRMs: IRM-19514.
Why it's important: Allows the system to add and maintain payment installments for Ad-hoc obligations, which is highly useful for handling inheritance taxes or migrated legacy debts.
16. RELIABILITY
Initiative: Reliability Enhancements
Key Epics/Features: Record Locking, Data Integrity in Distributed Transactions.
Relevant IRMs: IRM-36862, IRM-32912, IRM-17803.
Why it's important: Prevents data collisions by applying strict record-locking logic to Party records and improves error handling and rollback mechanisms in synchronous, distributed transactions.
17. DOCUMENT MANAGEMENT INTEGRATION
Initiative: Document Management Integration Enhancements
Key Epics/Features: Digital Signatures, Party Role Document Management.
Relevant IRMs: IRM-36668, IRM-34598, IRM-31308, IRM-28521, IRM-28303, IRM-28301.
Why it's important: Embeds 3rd-party digital signature tools and PDF previewers directly into RM. It also standardizes how documents are attached, categorized, and restricted (e.g., "Restricted to SS") across Party Roles.
18. SELF SERVICE SERVICES MANAGEMENT
Initiative: Self Service Services Management Enhancements
Key Epics/Features: Configurable Authentication, SS Party History, CMS Integration.
Relevant IRMs: IRM-34159, IRM-31771, IRM-31233, IRM-30785, IRM-30403.
Why it's important: Enhances the Self Service portal by allowing certain GET operations (like fetching reference data) without authentication. Also integrates CMS platforms and allows taxpayers to view their historical party details on demand.
19. OPERABILITY
Initiative: Operability Enhancements
Key Epics/Features: Call proxy services Documentation.
Relevant IRMs: IRM-32464.
Why it's important: Provides documentation and generic capabilities for SS to call backend APIs as operations, facilitating custom integrations and 3rd-party validations.
20. PENALTY & INTEREST
Initiative: Penalty & Interest Enhancements
Key Epics/Features: Credit Interest Calculation.
Relevant IRMs: IRM-29430.
Why it's important: Enhances the financial engine to calculate credit/reimbursement interest specifically for unallocated payment segments, ensuring taxpayers are properly credited for overpayments.
21. CONTACT MANAGEMENT
Initiative: Contact Management Enhancements
Key Epics/Features: Outbound Messages to Multiple Recipients, Unique Document IDs.
Relevant IRMs: IRM-33642, IRM-26951, IRM-26949, IRM-790.
Why it's important: Streamlines communication by allowing automated or manual batch-sending of messages to multiple recipients. It also ensures legal compliance by auto-generating unique IN/OUT/SYS reference IDs for all correspondence.
