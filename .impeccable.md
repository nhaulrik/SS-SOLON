## Design Context

### Users

**Primary users**: Developers, technical PMs, and power users who build AI-assisted presentation workflows. They are comfortable with JSON, understand prompt engineering, and work iteratively — uploading a PPTX template, tagging placeholders, feeding AI output back in, and repeating. They work at a desk, likely in a dimly lit office or home setup, focused on a single complex task. The tool is a professional instrument, not a consumer product.

**Job to be done**: Eliminate the tedious manual work of populating slide decks with AI-generated content. Users want a reliable, repeatable pipeline — not a flashy experience.

**Context**: Used in focused work sessions. Users are technically literate and expect precision. They care about correctness and workflow efficiency far more than visual delight.

### Brand Personality

**Three words**: Precise. Purposeful. Understated.

The interface should feel like a well-engineered instrument — the kind of tool a developer trusts because it gets out of the way. No marketing energy. No celebration. Just clarity and control.

**Emotional goal**: Confidence. Users should feel in control of a complex multi-step process, never lost, never second-guessing what the tool is doing.

### Aesthetic Direction

**Theme**: Dark. Users work in focused sessions, often at night or in low-light environments. The existing dark teal palette (`#061210` base, `#4CAF80` green accent) is already well-chosen for this context — it avoids the clichéd cyan/purple AI aesthetic and uses a nature-derived green that feels calm and deliberate rather than electric.

**Visual tone**: Technical-minimal. Tight, information-dense layouts are appropriate — this is a workflow tool, not a landing page. Whitespace should be purposeful, not decorative. The grid should be utilitarian.

**References**: Good engineering tools — Vercel dashboard, Linear, Warp terminal. Clean, dark, information-forward. Typography is functional. Hierarchy is created through weight and spacing, not decoration.

**Anti-references**: Anything that looks like a SaaS marketing site. No hero sections. No gradient text. No floating card grids. No sparklines used decoratively.

### Design Principles

1. **Hierarchy through density, not decoration.** The interface carries a lot of information (slide carousels, patch tables, recipe panels). Create clarity through typographic contrast and spatial rhythm — not borders, icons, or color splashes.

2. **Every element earns its place.** The existing design is already lean. New additions should justify themselves functionally. If something can be removed, it should be.

3. **Dark mode is the correct mode.** This is a tool used in focused, professional sessions. The dark teal palette is the right call. Do not introduce light-mode variants unless explicitly requested.

4. **Monospace for data, proportional for UI.** The recipe/JSON areas correctly use `JetBrains Mono`. The rest of the UI uses `Inter` — which is the right call for a dense tool UI, even though it's a reflex font. Given the existing codebase, replacing it would be disruptive without clear gain.

5. **Motion is information, not decoration.** The existing step-transition animations (`slideInRight`/`slideInLeft`) are well-conceived — they communicate direction. Any new motion should follow the same principle: it should tell the user something, not entertain them.
