Historical Accuracy
Enable the principle of historicity by maintaining the same code with different descriptions across non-overlapping validity periods for accurate historical tax processing.
03

Ensure no validity period overlaps exist for items with the same code within a codelist, supporting backdated taxpayer updates and amended return filing.
Key Epics & Features
Reference Data Enhancements
Enable codelist items to have the same code with different descriptions across non-overlapping validity periods, supporting EU tax code evolution (NACE, VAT Category, VATEX, Customs Codes).
Historical Code Management
Allow selection of historical codelist items when processing backdated taxpayer updates, maintaining historical data preservation for accurate reporting across different time periods.
Automatic Migration Support
Support automatic code updates when 1-to-1 mapping exists (e.g., AADE's KAD 2008 to KAD 2025 harmonization for 1.9 million Tax IDs as of March 1, 2026).
Manual Selection Interface
Provide manual selection interface when one code splits into multiple new codes, enabling taxpayers to select the most accurate code via the tax portal.
API & UI Enhancements
Implement API endpoints supporting querying codelist items by effective date and UI displays for historical versions of codelist items with proper validity period visualization.
Validation & Compliance
System validates no overlapping validity periods for same code within a codelist, ensuring temporal data integrity and regulatory compliance with EU Directive 2006/112/EC.
