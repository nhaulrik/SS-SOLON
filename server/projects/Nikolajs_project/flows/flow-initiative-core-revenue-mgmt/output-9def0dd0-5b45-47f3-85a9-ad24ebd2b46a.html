Why It's Important

Configuration Alignment
By enabling admins to configure separate bill segment types for source and target segments in the Transfer Amount process, we ensure alignment with existing, predefined bill segment configurations used across other procedures.
02
Reduced Duplication
Reduces configuration duplication, minimizes manual corrections, and improves consistency across revenue types and bill segment categories.
03
Ad-hoc Obligation Management
Enables users to create multiple payment obligation actions under ad-hoc obligations as installments, following the structure of non ad-hoc obligations with installments for Inheritance tax and legacy data migration processes.
Key Epics & Features
Transfer Amount - Enhancement on configuration of Bill segment types
As an admin, configure the bill segment types generated by the Transfer Amount process to align with existing, predefined bill segment configurations. Includes new Alternative Target Bill segment type field and relabeling of existing field as Default Bill segment type with mandatory validation and tooltip guidance.
Add and Maintain Ad-hoc Obligations – Pay Instalment Actions and Categories
Enable users and admins to create multiple payment obligation actions under ad-hoc obligations as installments. Users can add Filing and multiple Payment Actions flagged as Installments, define categories (Preliminary, Original), set due dates, and view comprehensive lists with sorting and detail updates.
Bill Segment Type Configuration Mechanism
Default bill segment type is mandatory and applies to source bill segment. Alternative Target Bill segment type applies to target bill segment. If Alternative Target Bill segment type is not filled, Default bill segment type is used for target bill segment.
Ad-hoc Obligation Action Management
Users can see lists of pay instalment actions with identifying information including action category, action type, reference number, due date, amount, and action status. Users can create manually, sort lists, and open actions to view details and update as needed.
Advanced Installment Creation Rules
Predefined rules enable users to create multiple installments by adding parameters such as start date, due date, number of installments, duration between installments, action type, and grace date margin. Admins can define rules in revenue type configuration.
Non-Functional Requirements
All solutions must maintain and improve Performance, Security, Accessibility, and Usability. All Definitions of Done must be fulfilled with proper documentation of adjustments and collaboration with specialists.

CREDIT_INT Rule Documentation
Document all adjustments for penalty and interest rule CREDIT_INT including performance, security, traceability, accessibility, and usability non-functional requirements.

Why It's Important
01
Consolidated Financial View
Provides Revenue Authority users with a clear, consolidated view of all financial activities within a specific reconciliation period for a taxpayer's Tax Account, improving efficiency and accuracy.
02
Period-Centric Overview
Aggregates and presents all financial transactions related to selected reconciliation periods for a given revenue type registration, providing an immediate consolidated view of that period's financial status.
03
Enhanced Visibility
Bridges the gap in Solon Tax by introducing a dedicated user interface that presents Tax Account information in a single, easily accessible view for improved visibility and efficiency.
Key Epics & Features
Tax Account Overview Navigation
New redirect link on Revenue Type Registration page visible only for RTRs configured as Tax Account, navigating users to the Tax Account Overview page with current RTR context and selected reconciliation period.
Reconciliation Period Filtering
Users can filter financial transactions by reconciliation period with configurable From and To dates, supporting selection of multiple reconciliation periods for comprehensive period analysis.
Financial Transactions Table
Read-only table displaying all Financial Transactions linked to selected reconciliation period with opening and closing balances, supporting sorting, pagination, and Column Configurator functionality.
Opening and Closing Balance Calculation
Opening Balance equals closing balance of previous period; Closing Balance equals Opening plus all financial transactions including debts, payments, interests, write-offs, and refunds for the period.
Update Penalty & Interest Button
Users can access and select the Update P&I button to enter RECALCULATEFROMDATE parameter for running Penalty & Interest calculations on the Tax Account.
Comprehensive Transaction Columns
Table includes Creation Date, Effective Date, Accounting Date, Origin, FT ID, Revenue Type, FT Type, Posting Reason, Liability Balance, and Running Balance for complete financial transaction visibility.
