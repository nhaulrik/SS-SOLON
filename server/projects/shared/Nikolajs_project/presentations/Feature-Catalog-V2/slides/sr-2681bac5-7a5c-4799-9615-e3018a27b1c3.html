Why It's Important
01
Data Privacy & Compliance
Enabling administrators to restrict access to sensitive personal data based on defined classifications strengthens data privacy, compliance, and governance aligned with GDPR and regulatory requirements.
02
Granular Access Control
Allows granular control over who can read, update, or delete personal data—either via the UI or APIs—enabling enforcement of data protection policies across all access channels.
03
Risk Mitigation
Reduces the risk of unauthorized data access by ensuring sensitive details are not inadvertently exposed in logs, messages, reports, or other data sources, even during exception handling or auditing.
Key Epics & Features
Data Security - Restrict Access to Personal Data GDPR (Part 1/4)
Admin can restrict access to sensitive personal data based on classification level and group of users. Enforce permissions for Create, Read, Update, Delete, and Search operations via UI, direct service calls, or APIs with Data Access Layer enforcement.
Data Security - Define Access per User Access Group (Part 2/4)
Admin can define permissions per classification level per user access groups. UI available in RM for admin to configure permission levels (Read, Search, Update) with masking or non-visible modes.
Data Security - Restrict Access in Logs (Part 3/4)
Admin can remove or mask access to sensitive personal data types in Security, Audit, and Access logs. Manage sensitive data in audit logs to prevent exposure of classified information during exception handling.
Cross-Entity Data Propagation
Restrictions propagate to all entities displaying or using classified data including Forms, Cases, Human Tasks, Taxpayer Accounts, Registration Records, Users, Notifications, Contacts, and Notes.
Identification Labels & Message Store
Identification labels display according to classification restrictions with masking or non-visibility modes. Admin can restrict access to message store records based on classification levels and permissions.
Operational Reports & NFR Testing
Visibility of records in operational reports restricted based on classification levels and permissions. Ensure Performance, Security, Accessibility, and Usability NFRs are tested without negative impact on product.
