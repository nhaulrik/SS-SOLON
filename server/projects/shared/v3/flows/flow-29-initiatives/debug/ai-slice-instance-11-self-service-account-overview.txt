Issue Type: Feature | IRM: IRM-16516 | Roadmap Initiative Group: REFERENCE IMPLEMENTATIONS | Roadmap Initiative: SELF SERVICE ACCOUNT OVERVIEW | Roadmap Feature: SS  Payments - See Taxpayer Account Overview - User - Part 3 - Open/Edit Registration Details | Requester: Other | PI: PI29 | Estimate: 74.9 | Feature Priority: p3 | Feature Status: Waiting Analysis Approval | Features Description: As a self service user, I want to see an overview of a taxpayer account that I can access, so that I can confirm this is the taxpayer account that I intended to access, can understand the obligations for this account, can see notifications sent to this account and see the taxpayer account balances.

*Introduction*

This feature is related to IRM-3447 in which requirements have been doccumented to see an overview of a taxpayer account on the self service.

Some example wireframes have been attached to this feature.  In case the content on the wireframes differs from the functional requirements, then the functional requirement should be used as input when implementing this feature.

*Functional requirements*
 * User can see registration details of the taxpayer.
 ** {color:#c1c7d0}Data set:{color}
 *** {color:#c1c7d0}Name: the primary name of the taxpayer for which the account overview is shown{color}
 *** {color:#c1c7d0}ID type: the type of the primary identifier of the taxpayer for which the account overview is shown{color}
 *** {color:#c1c7d0}ID number: the primary identifier number of the taxpayer for which the account overview is shown{color}
 *** {color:#c1c7d0}Address: the preferred address for communication with the taxpayer for which the account overview is shown{color}
 *** {color:#c1c7d0}Email: the preferred email address for communication with the taxpayer for which the account overview is shown{color}
 ** Actions:
 *** {color:#172b4d}User can open the registration details{color}
 *** {color:#172b4d}User can open and update the registration details{color}
 * {color:#c1c7d0}User can see the current balances for each tax type for which this taxpayer was registered{color}
 ** {color:#c1c7d0}Data set - can contain data for multiple tax types{color}
 *** {color:#c1c7d0}Tax type: a tax type for which this taxpayer was registered{color}
 *** {color:#c1c7d0}Balance: the current balance for a tax type for which a taxpayer was registered - any period in which the tax type was applicable for this taxpayer should be considered when calculating the balance{color}
 ** {color:#c1c7d0}Actions:{color}
 *** {color:#c1c7d0}User can open the details for a specific tax type for which this taxpayer was registered (see IRM-3449){color}
 * {color:#c1c7d0}User can see the past, present, and future obligations and their status for the taxpayer account in a timeline{color}
 ** {color:#c1c7d0}Data set - can contain data for multiple obligations{color}
 *** {color:#c1c7d0}Tax type: the tax type for which the obligation exists{color}
 *** {color:#c1c7d0}Obligation action type: the type of action that the taxpayer is obliged to take{color}
 *** {color:#c1c7d0}Period: the period for which the obligation exists (if any){color}
 *** {color:#c1c7d0}Status: the status of the action that the taxpayer is obliged to take - user can for example see if the action has been completed or not - and see for actions that have not been completed yet whether the action is considered overdue or not (e.g. overdue is red, due is orange and completed is green){color}
 *** {color:#c1c7d0}Obligation Due date: the date by which the action, that the taxpayer is obliged to take, must be completed{color}
 ** {color:#c1c7d0}Actions:{color}
 *** {color:#c1c7d0}User can open the details for a specific obligation{color} (see IRM-3451)
 * {color:#c1c7d0}User can see the latest notifications{color}
 ** {color:#c1c7d0}Data set - can contain data for multiple notifications{color}
 *** {color:#c1c7d0}Notification date: the date the notification was sent to the taxpayer{color}
 *** {color:#c1c7d0}Notification message: the notification message header{color}
 *** {color:#c1c7d0}Notification read or unread: indication whether the user has read the message ate least once or not{color}
 *** {color:#c1c7d0}Notification importance: the importance of the notification{color}
 ** {color:#c1c7d0}Actions:{color}
 *** {color:#c1c7d0}User can open the details for a specific notification{color}
