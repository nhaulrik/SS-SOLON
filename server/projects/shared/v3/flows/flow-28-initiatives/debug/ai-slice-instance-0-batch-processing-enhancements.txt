Overview

This initiative modernizes the Amplio batch processing system by enhancing maintainability, resilience, and operational flexibility. Building on learnings from the Lithuania project and the migration of batch jobs to Amplio the work focuses on refactoring existing batch jobs rather than creating new features.
h2. Key Objectives
 * Standardize project structure and package naming (replace {{...etl...}} with {{{}...batch.batchjob...{}}})
 * Centralize duplicated configurations (Keycloak credentials, DataSource, FeignClient beans)
 * Resolve artifact naming conflicts between batch and batch-custom projects
 * Clean up redundant POM dependencies

h2. Business Value
 * Improved system resilience and reduced manual intervention
 * Reduced technical debt and improved long-term maintainability
 * More stable and secure codebase
