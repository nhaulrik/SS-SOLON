Issue Type: Feature | IRM: IRM-38379 | Roadmap Initiative Group: CORE SUPPORTING CAPABILITIES | Roadmap Initiative: FORM PROCESSING | Roadmap Feature: Forms : Be able to log details for the submitter of the form /RM &SS | Requester: GRC | PI: PI28 | Estimate: 373.1 | Feature Priority: p1 | Feature Status: In Design | Features Description: As an Administrator, I need the ability to record the user responsible for creating and submitting a form, as in many cases this user may not be the same as the Party associated with the form.

 

 

*Functional Requirements* 

*a)* Optional fields in Form metadata
*Created By* : Reference to a Party Role ID representing the user who created the form
*Posted By* : Reference to a Party Role ID representing the user who submitted/posted the form
*External System Created Date/Time* (external_created_dttm) :Timestamp indicating when the form was created in an external system (similar to the Party record field)

*b)* user may search forms using the above fields.

*c)* Enable user-define-characteristics (Data Area)  in forms for other metadata related items. (Data Area = Form Details)  
Example: Import Date from external system. (Date field)

*d)* Apply Notes in Forms in RM and SS 
In UI add Notes only in RM

 

*Documentation*
Document all the above adjustments. 

 

*Non functional requirements*
It is expected that the NFRs are considered and examined: Performance , Security, Accessibility, Usability 

DODs should be fulfilled.
