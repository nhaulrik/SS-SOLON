Issue Type: Feature | IRM: IRM-790 | Roadmap Initiative Group: CORE SUPPORTING CAPABILITIES | Roadmap Initiative: CONTACT MANAGEMENT | Roadmap Feature: Send Outbound Message Automatically to Multiple Recipients - Admin | Requester: GRC | PI: PI28 | Estimate: 38.4 | Feature Priority: p3 | Feature Status: Waiting Analysis Approval | Features Description: As an admin, I want to automatically send a copy of a specific outbound message (e.g. letter, email or text message) to different types of recipients (e.g. taxpayer and tax agent), so that the information in the outbound message can be exchanged with all relevant parties.

*Introduction*

This feature extends IRM-786 and IRM-789 where requirements have been documented to send an outbound letter and outbound email / text messages automatically to a single party.

*Functional Requirements*

Admin can enter literal values and/or reference values that will be known at run-time for input parameters that admin has defined for this type of contact and that can be used to automatically select the type of recipients that need to be considered for this type of outbound message, and automatically select a recipient for each type of recipient (based on rules defined by admin). These values can be defined on a 'establish and/or record contact' task within a specific type of business process (see IRM-785).

When executing the 'establish and/or record contact' task on an instance of this type of business process, the system will automatically generate all outbound messages, store these messages, finalize the messages, select a communication channel for each recipient and send the finalized outbound messages via the selected communication channels (e.g. print letter or send email).
Issue Type: Feature | IRM: IRM-775 | Roadmap Initiative Group: CORE SUPPORTING CAPABILITIES | Roadmap Initiative: CONTACT MANAGEMENT | Roadmap Feature: Define and Maintain Contact Types - Select Recipients - Admin | Requester: GRC | PI: PI28 | Estimate: 42 | Feature Priority: p3 | Feature Status: Waiting Analysis Approval | Features Description: As an admin, I want to define and maintain rules to determine what recipient(s) need to receive a copy of a certain type of outbound message (e.g. certain type of letter, email or SMS), so that duplicate messages can be send to all parties that need to be informed.

*Introduction*

This feature extends IRM-772 in which requirements have been documented to define and maintain different types of contacts.

Recipients will receive the same information or almost the same information. For example, one letter could contain text to explain that this letter is a copy of the letter that has been sent to the taxpayer representative.
Relevant to https://jira.intrasoft-intl.com/browse/IRM-774 

*Functional Requirements*
 * Admin can define rules to determine what types of recipients must receive a copy of a specific type of outbound message under certain circumstances. For example, if a certain type of letter is sent to the recipient type 'taxpayer' then a copy of this letter may have to be sent to the recipient type 'tax agent' and/or recipient type 'spouse'.
 * Admin can define rules to determine the recipient for each type of recipient that will receive a copy of the message. For example, find the 'tax agent' that is representing a 'taxpayer' for a specific type of tax. Input parameters defined on the contact type can be used to select the recipients. Values for these input parameters can be provided at design time (literal values) or can be provided at run-time.
