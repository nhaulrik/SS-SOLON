Record Locking - Apply logic to Party record

Concurrency Conflict Handling
Implements mismatch detection and rejection logic. On version mismatch, transaction is rejected with message: "This record was modified by another user. Please reload and review the latest data." UI users reload and retry; batch/BPMN processes execute retry policy with configurable attempts.
