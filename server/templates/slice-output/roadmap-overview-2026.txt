=== DATA EXTRACTION RULES (read before filling any field) ===

SCOPE:
- This template covers the full 2026 roadmap (all PIs). It is NOT filtered to a single PI.
- Year-level totals (Spent, Planned, Total) come from the "Reference Sheet".
- PI-level total hours come from the "Reference Sheet".
- Per-PI requester breakdowns (GRC / GTM / Other hour values) come from the "Requester Stats" sheet.
- Per-PI initiative group breakdowns come from the "Initiative Group Stats" sheet.

REFERENCE SHEET LOOKUP RULE:
- Sheet name: Reference Sheet
- Column B = label, Column C = value (numeric)
- Match on the exact label text in column B. Read the value from column C.
- This is a direct cell read — do NOT sum or derive values from other sheets.

REQUESTER STATS LOOKUP RULE:
- Sheet name: Requester Stats
- Columns: PI | Requester | Estimate
- To get a value, find the single row matching both PI and Requester. Read the Estimate cell directly.
- Valid Requester values: "GRC", "GTM", "Other"
- If no row exists for a given PI + Requester combination, treat the value as 0 and write "-" for display.

LOOKUP DISCIPLINE — NO SUMMATION:
- Do NOT sum rows from any sheet to derive a total. Every numeric value must be a direct cell read.
- The only exception is percentages — those are calculated by the AI as described per field.
- If a matching row cannot be found, write N/A unless the field says to write "-".

HOURS FORMATTING RULE:
- Display all hours in "k" notation.
- Round to one decimal place, then drop trailing ".0": 56218 → "56.2k", 85233 → "85.2k", 160808 → "160.8k".
- For the large headline number (.scope-big), round to nearest whole k: 246041 → "246k".
- Percentages: round to nearest integer, e.g. 41.6% → "42%".

BAR PERCENTAGE RULE (initiative group rows inside PI cards):
- For each PI card, find the initiative group with the highest Estimate among all groups in that PI.
- That group receives bar width: 100%.
- All other groups: round( (group_estimate / max_estimate_in_pi) × 100 ) → integer.
- Write result as e.g. "47%" (include the % sign).

=== REQUESTER DISTRIBUTION (full-year backlog) ===

Per-requester hour values: read from Requester Stats — one row per Requester across all PIs.
Full-year totals are NOT pre-computed in the sheet; sum the Estimate column per Requester across all PI rows.

  GRC_TOTAL:   sum of Estimate where Requester = "GRC"    →  58322  →  "58.3k"
  GTM_TOTAL:   sum of Estimate where Requester = "GTM"    →  4415   →  "4.4k"
  OTHER_TOTAL: sum of Estimate where Requester = "Other"  →  101297 →  "101.3k"

  REQUESTER_SUM = GRC_TOTAL + GTM_TOTAL + OTHER_TOTAL

  Percentages (AI-computed):
    GRC_PCT   = round( GRC_TOTAL   / REQUESTER_SUM × 100 )  →  36%
    GTM_PCT   = round( GTM_TOTAL   / REQUESTER_SUM × 100 )  →  3%
    OTHER_PCT = round( OTHER_TOTAL / REQUESTER_SUM × 100 )  →  62%

.req-title text:
  "Feature Backlog by Requester (160.8k)"

Requester bar widths:
  .req-seg.grc  style.width → GRC_PCT   + "%" and inner text → GRC_PCT   + "%"
  .req-seg.gtm  style.width → GTM_PCT   + "%" and inner text → GTM_PCT   + "%"
  .req-seg.oth  style.width → OTHER_PCT + "%" and inner text → OTHER_PCT + "%"

Legend (.req-leg-pct):
  1st (GRC)   → "58.3k"
  2nd (GTM)   → "4.4k"
  3rd (Other) → "101.3k"
